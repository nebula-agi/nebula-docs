# Quick Start Guide

<Warning>You'll need a Nebula API key before starting. Get one at [trynebula.ai](https://trynebula.ai)</Warning>

Get up and running with Nebula in minutes.

## Prerequisites

- Python 3.8+ or Node.js 16+
- A Nebula account at [trynebula.ai](https://trynebula.ai)

## 1. Install

```bash
# Python
pip install nebula-client

# Node.js
npm install @nebula-ai/sdk
```

## 2. Initialize Client

<CodeGroup>

```python Python
from nebula import Nebula

# With API key
nebula = Nebula(api_key="your-api-key-here")

# Or set NEBULA_API_KEY environment variable
# nebula = Nebula()
```

```javascript JavaScript
const { Nebula } = require('@nebula-ai/sdk');

const nebula = new Nebula({
  apiKey: 'your-api-key-here'
});
```

```bash cURL
# Verify your API key by listing collections
curl -X GET "https://api.trynebula.ai/v1/collections?limit=5" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</CodeGroup>

## 3. Store & Search

<CodeGroup>

```python Python
from nebula import Nebula

nebula = Nebula(api_key="your-api-key-here")

# Create a collection
collection = nebula.create_collection(name="my_notes", description="Personal notes")

# Store a memory
memory_id = nebula.store_memory({
    "collection_id": collection.id,
    "content": "Machine learning is transforming healthcare",
    "metadata": {"topic": "AI", "importance": "high"}
})

# Search memories
results = nebula.search(
    query="machine learning healthcare",
    collection_ids=[collection.id]
)

# Results contain entities, facts, and utterances
for entity in results.entities:
    print(entity)
for fact in results.facts:
    print(fact)
```

```javascript JavaScript
const { Nebula } = require('@nebula-ai/sdk');

const nebula = new Nebula({
  apiKey: 'your-api-key-here'
});

async function storeAndSearch() {
  // Create a collection
  const collection = await nebula.createCollection({
    name: 'my_notes',
    description: 'Personal notes'
  });

  // Store a memory
  const memoryId = await nebula.storeMemory({
    collection_id: collection.id,
    content: 'Machine learning is transforming healthcare',
    metadata: { topic: 'AI', importance: 'high' }
  });

  // Search memories
  const results = await nebula.search({
    query: 'machine learning healthcare',
    collection_ids: [collection.id]
  });

  // Results contain entities, facts, and utterances
  results.entities?.forEach(entity => console.log(entity));
  results.facts?.forEach(fact => console.log(fact));
}

storeAndSearch();
```

```bash cURL
# 1) Create a collection
curl -X POST "https://api.trynebula.ai/v1/collections" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"name": "my_notes", "description": "Personal notes"}'

# 2) Store a memory (use collection ID from step 1)
curl -X POST "https://api.trynebula.ai/v1/memories" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "collection_id": "COLLECTION_ID",
    "engram_type": "document",
    "raw_text": "Machine learning is transforming healthcare",
    "metadata": {"topic": "AI", "importance": "high"}
  }'

# 3) Search memories
curl -X POST "https://api.trynebula.ai/v1/memories/search" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "query": "machine learning healthcare",
    "collection_ids": ["COLLECTION_ID"]
  }'
```

</CodeGroup>

## What You'll Get Back

When you store a memory, Nebula returns the memory ID:

```python
memory_id = "eng_abc123def456"
```

When you search, Nebula returns a `MemoryResponse` with three layers of structured knowledge:

```json
{
  "entities": [
    {
      "name": "Machine Learning",
      "description": "A subset of AI focused on learning from data",
      "activation_score": 0.95
    }
  ],
  "facts": [
    {
      "subject": "Machine Learning",
      "predicate": "is transforming",
      "value": "healthcare",
      "activation_score": 0.92
    }
  ],
  "utterances": [
    {
      "text": "Machine learning is transforming healthcare",
      "activation_score": 0.91,
      "engram_id": "eng_abc123def456"
    }
  ]
}
```

- **Entities**: People, concepts, and things extracted from your data
- **Facts**: Subject-predicate-value assertions about those entities
- **Utterances**: The original source text that grounds each fact

<Tip>See [Core Concepts](/guides/concepts) for a deeper explanation of entities, facts, and utterances.</Tip>
