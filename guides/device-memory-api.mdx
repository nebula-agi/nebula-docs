<Info>Device memory reuses the existing `store_memory` and `search` endpoints. Pass `snapshot` instead of `collection_id` -- they are mutually exclusive.</Info>

All endpoints require authentication via `X-API-Key` header (or `Authorization: Bearer <token>`).

## Export Snapshot

Export a collection's full graph state as a portable snapshot.

```
POST /v1/device-memory/snapshot/export
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `collection_id` | string | Yes | UUID of the collection to export |

<CodeGroup>

```python Python
from nebula import Nebula

client = Nebula(api_key="YOUR_API_KEY")
snapshot = client.export_snapshot(collection_id="YOUR_COLLECTION_ID")
```

```python Python (async)
from nebula import AsyncNebula

async with AsyncNebula(api_key="YOUR_API_KEY") as client:
    snapshot = await client.export_snapshot(collection_id="YOUR_COLLECTION_ID")
```

```typescript JavaScript
import { Nebula } from '@nebula-ai/sdk';

const client = new Nebula({ apiKey: 'YOUR_API_KEY' });
const snapshot = await client.exportSnapshot('YOUR_COLLECTION_ID');
```

```bash cURL
curl -X POST "https://api.trynebula.ai/v1/device-memory/snapshot/export" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{"collection_id": "YOUR_COLLECTION_ID"}'
```

</CodeGroup>

## Store Memory

The existing `/v1/memories` endpoint accepts a `snapshot` field instead of `collection_id`. When a snapshot is provided, Nebula processes the content statelessly and returns an updated snapshot instead of a `memory_id`.

```
POST /v1/memories
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `snapshot` | object | Yes | Your current snapshot (mutually exclusive with `collection_id`) |
| `raw_text` | string | Yes | Content to store |
| `metadata` | object | No | Arbitrary metadata |

Returns `{ "snapshot": <updated_snapshot> }` when using device memory mode.

<CodeGroup>

```python Python
from nebula import Memory

result = client.store_memory(
    Memory(
        snapshot=snapshot,
        content="Had a meeting with Alice about the Q2 roadmap",
    )
)

snapshot = result.snapshot
```

```python Python (async)
async with AsyncNebula(api_key="YOUR_API_KEY") as client:
    result = await client.store_memory(
        Memory(
            snapshot=snapshot,
            content="Had a meeting with Alice about the Q2 roadmap",
        )
    )
    snapshot = result.snapshot
```

```typescript JavaScript
const result = await client.storeMemory({
  snapshot,
  content: 'Had a meeting with Alice about the Q2 roadmap',
});

snapshot = result.snapshot;
```

```bash cURL
curl -X POST "https://api.trynebula.ai/v1/memories" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "snapshot": { "...your snapshot..." },
    "raw_text": "Had a meeting with Alice about the Q2 roadmap"
  }'
```

</CodeGroup>

## Search

The existing `/v1/memories/search` endpoint accepts a `snapshot` field instead of `collection_ids`. Nebula scores your snapshot statelessly and returns results. Nothing is persisted.

```
POST /v1/memories/search
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `snapshot` | object | Yes | Your full snapshot (mutually exclusive with `collection_ids`) |
| `query` | string | Yes | Natural language search query |
| `limit` | integer | No | Maximum results to return (default: 10) |

Returns the same `MemoryResponse` format as standard search (semantics, procedures, episodes, sources).

<CodeGroup>

```python Python
results = client.search(
    query="What do I know about Alice?",
    snapshot=snapshot,
    limit=10,
)

for fact in results.semantics:
    print(f"{fact['subject']} -> {fact['predicate']} -> {fact['value']}")
```

```python Python (async)
results = await client.search(
    query="What do I know about Alice?",
    snapshot=snapshot,
    limit=10,
)
```

```typescript JavaScript
const results = await client.search({
  query: 'What do I know about Alice?',
  snapshot,
  limit: 10,
});

for (const fact of results.semantics) {
  console.log(`${fact.subject} -> ${fact.predicate} -> ${fact.value}`);
}
```

```bash cURL
curl -X POST "https://api.trynebula.ai/v1/memories/search" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "snapshot": { "...your snapshot..." },
    "query": "What do I know about Alice?",
    "limit": 10
  }'
```

</CodeGroup>

## Import Snapshot

Import a snapshot into an ephemeral server-side collection. Useful for debugging or running server-side operations against a client-owned snapshot.

```
POST /v1/device-memory/snapshot/import
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `snapshot` | object | Yes | The full snapshot to import |

Returns `{ "ephemeral_collection_id": "<uuid>" }`.

<CodeGroup>

```python Python
ephemeral_id = client.import_snapshot(snapshot=snapshot)
```

```python Python (async)
ephemeral_id = await client.import_snapshot(snapshot=snapshot)
```

```typescript JavaScript
const ephemeralId = await client.importSnapshot(snapshot);
```

```bash cURL
curl -X POST "https://api.trynebula.ai/v1/device-memory/snapshot/import" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{"snapshot": { "...your snapshot..." }}'
```

</CodeGroup>
