# R2R API Reference

Welcome to the comprehensive API reference for the R2R (Retrieval-Augmented Generation) platform. This documentation covers all endpoints available in the R2R API v3.

## Overview

The R2R API is organized into 13 main categories, each handling different aspects of the platform:

- **System** - Core system operations and health checks
- **Users** - User management, authentication, and authorization
- **Collections** - Document collection management
- **Documents** - Document ingestion, management, and processing
- **Chunks** - Text chunk management and operations
- **Indices** - Search index management
- **Conversations** - Chat and conversation management
- **Retrieval** - Search and RAG operations
- **Prompts** - Prompt template management
- **Graph** - Knowledge graph operations and entity management
- **Episodes** - Conversation episode management
- **Contradictions** - Contradiction detection and management
- **Temporal Events** - Time-based event management

## Authentication

All API endpoints require authentication using API keys or user credentials. Include your API key in the Authorization header:

```bash
Authorization: Bearer YOUR_API_KEY
```

## Base URL

The base URL for all API endpoints is:

```
https://api.r2r.ai/v3
```

## Rate Limiting

API requests are subject to rate limiting. Check the response headers for rate limit information:

- `X-RateLimit-Limit`: Maximum requests per time window
- `X-RateLimit-Remaining`: Remaining requests in current window
- `X-RateLimit-Reset`: Time when the rate limit resets

## Response Format

All API responses follow a consistent format:

```json
{
  "success": true,
  "data": {
    // Response data
  },
  "message": "Success message"
}
```

## Error Handling

Errors are returned with appropriate HTTP status codes and error messages:

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message"
  }
}
```

1. **Get an API Key**: Sign up at [r2r.ai](https://r2r.ai) to get your API key
2. **Choose your SDK**: Install the Python or JavaScript SDK
3. **Make your first request**: Start with a simple health check or document upload

## Examples

### Python Example

```python
from r2r import R2RClient

# Initialize client
client = R2RClient(api_key="your-api-key")

# Upload a document
response = client.documents.create(
    file_path="document.pdf",
    metadata={"title": "My Document"}
)

# Search documents
results = client.retrieval.search(
    query="What is machine learning?",
    limit=10
)
```

### JavaScript Example

```javascript
const { R2RClient } = require('r2r-js');

// Initialize client
const client = new R2RClient('your-api-key');

// Upload a document
const response = await client.documents.create({
    file: { path: 'document.pdf', name: 'document.pdf' },
    metadata: { title: 'My Document' }
});

// Search documents
const results = await client.retrieval.search({
    query: 'What is machine learning?',
    limit: 10
});
```
