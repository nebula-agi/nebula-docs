# System Operations

The System API provides core system operations including health checks, configuration management, and system status monitoring.

## Endpoints

### Health Check

Check the health status of the R2R system.

<RequestExample>
```bash
curl -X GET "https://api.r2r.ai/v3/health" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

<ResponseExample>
```json
{
  "success": true,
  "data": {
    "status": "healthy",
    "timestamp": "2024-01-15T10:30:00Z",
    "version": "1.0.0"
  }
}
```
</ResponseExample>

**Endpoint:** `GET /health`

**Description:** Returns the current health status of the R2R system, including system status, timestamp, and version information.

**Authentication:** Not required

**Response Fields:**
- `status` (string): System health status ("healthy", "degraded", "unhealthy")
- `timestamp` (string): ISO 8601 timestamp of the health check
- `version` (string): Current system version

---

### Get Configuration

Retrieve the current system configuration.

<RequestExample>
```bash
curl -X GET "https://api.r2r.ai/v3/config" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

<ResponseExample>
```json
{
  "success": true,
  "data": {
    "app_name": "R2R API",
    "version": "1.0.0",
    "environment": "production",
    "features": {
      "auth_enabled": true,
      "rate_limiting": true,
      "logging": true
    },
    "limits": {
      "max_file_size": "100MB",
      "max_concurrent_requests": 100
    }
  }
}
```
</ResponseExample>

**Endpoint:** `GET /config`

**Description:** Returns the current system configuration including feature flags, limits, and environment settings.

**Authentication:** Required

**Response Fields:**
- `app_name` (string): Application name
- `version` (string): System version
- `environment` (string): Environment (production, staging, development)
- `features` (object): Feature flags and capabilities
- `limits` (object): System limits and constraints

---

### Get System Status

Get detailed system status and metrics.

<RequestExample>
```bash
curl -X GET "https://api.r2r.ai/v3/status" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

<ResponseExample>
```json
{
  "success": true,
  "data": {
    "system": {
      "uptime": "24h 30m 15s",
      "memory_usage": "75%",
      "cpu_usage": "45%",
      "disk_usage": "60%"
    },
    "services": {
      "database": "healthy",
      "vector_store": "healthy",
      "embedding_service": "healthy",
      "llm_service": "healthy"
    },
    "metrics": {
      "total_requests": 15420,
      "active_users": 125,
      "documents_processed": 8920
    }
  }
}
```
</ResponseExample>

**Endpoint:** `GET /status`

**Description:** Returns detailed system status including uptime, resource usage, service health, and performance metrics.

**Authentication:** Required

**Response Fields:**
- `system` (object): System-level metrics
  - `uptime` (string): System uptime
  - `memory_usage` (string): Memory usage percentage
  - `cpu_usage` (string): CPU usage percentage
  - `disk_usage` (string): Disk usage percentage
- `services` (object): Health status of individual services
- `metrics` (object): Performance and usage metrics

## Error Codes

| Code | Description |
|------|-------------|
| `401` | Unauthorized - Invalid or missing API key |
| `403` | Forbidden - Insufficient permissions |
| `500` | Internal Server Error - System error |

## Rate Limiting

System endpoints are subject to rate limiting:
- Health check: 100 requests per minute
- Configuration and status: 10 requests per minute

## SDK Examples

### Python

```python
from r2r import R2RClient

client = R2RClient(api_key="your-api-key")

# Check system health
health = client.system.health()
print(f"System status: {health['status']}")

# Get configuration
config = client.system.config()
print(f"Version: {config['version']}")

# Get system status
status = client.system.status()
print(f"Uptime: {status['system']['uptime']}")
```

### JavaScript

```javascript
const { R2RClient } = require('r2r-js');

const client = new R2RClient('your-api-key');

// Check system health
const health = await client.system.health();
console.log(`System status: ${health.status}`);

// Get configuration
const config = await client.system.config();
console.log(`Version: ${config.version}`);

// Get system status
const status = await client.system.status();
console.log(`Uptime: ${status.system.uptime}`);
```
