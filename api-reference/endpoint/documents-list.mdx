# List Documents

Retrieve a list of documents accessible to the authenticated user.

<RequestExample>
```bash
curl -X GET "https://api.r2r.ai/v3/documents?offset=0&limit=10&owner_only=false" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

<ResponseExample>
```json
{
  "success": true,
  "data": {
    "results": [
      {
        "id": "b4ac4dd6-5f27-596e-a55b-7cf242ca30aa",
        "title": "Sample Document",
        "content": "This is the document content...",
        "metadata": {
          "author": "John Doe",
          "created_date": "2024-01-15"
        },
        "created_at": "2024-01-15T10:30:00Z",
        "updated_at": "2024-01-15T10:30:00Z"
      }
    ],
    "total": 1,
    "offset": 0,
    "limit": 10
  }
}
```
</ResponseExample>

**Description:** Returns a paginated list of documents that the authenticated user has access to. Users can see documents they own and documents in collections they are members of.

**Authentication:** Required

**Query Parameters:**
- `ids` (array of strings, optional): A list of document IDs to retrieve. If not provided, all accessible documents will be returned.
- `offset` (integer, optional, default: 0): Number of objects to skip. Must be >= 0.
- `limit` (integer, optional, default: 100): Limit on the number of objects to return. Must be between 1 and 1000.
- `include_summary_embeddings` (boolean, optional, default: false): Whether to include embeddings of each document summary.
- `owner_only` (boolean, optional, default: false): If true, only returns documents owned by the user, not all accessible documents.

**Response Fields:**
- `results` (array): Array of document objects
  - `id` (string): Unique identifier for the document
  - `title` (string): Document title
  - `content` (string): Document content
  - `metadata` (object): Document metadata
  - `created_at` (string): ISO 8601 timestamp when the document was created
  - `updated_at` (string): ISO 8601 timestamp when the document was last updated
- `total` (integer): Total number of documents
- `offset` (integer): Current offset
- `limit` (integer): Current limit

## Error Codes

| Code | Description |
|------|-------------|
| `400` | Bad Request - Invalid query parameters |
| `401` | Unauthorized - Invalid or missing API key |
| `500` | Internal Server Error - System error |

## SDK Examples

### Python

```python
from r2r import R2RClient

client = R2RClient(api_key="your-api-key")

# List documents
result = client.documents.list(
    offset=0,
    limit=10,
    owner_only=False
)
print(f"Found {result['total']} documents")
```

### JavaScript

```javascript
const { r2rClient } = require("r2r-js");

const client = new r2rClient("your-api-key");

// List documents
const response = await client.documents.list({
    offset: 0,
    limit: 10,
    ownerOnly: false
});
console.log(`Found ${response.total} documents`);
```
