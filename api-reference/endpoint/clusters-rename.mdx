Rename a cluster within a project. Requires authentication and appropriate permissions (owner or project admin).

## Endpoint

`POST /clusters/rename`

## Description
Renames a cluster by name or ID in the authenticated user's project. Only the cluster owner or a project admin can rename a cluster. The new name must be unique within the project.

## Request

**Headers:**
- `Authorization: Bearer <token>` or `Authorization: ApiKey <api_key>`

**Body:**
```json
{
  "cluster": "string", // Required. Cluster name or ID.
  "newName": "string"   // Required. New unique name for the cluster.
}
```

## Example cURL

```bash
curl -X POST https://api.nebulacloud.app/clusters/rename \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "cluster": "my-cluster",
    "newName": "renamed-cluster"
  }'
```

## Success Response

**Status:** `200 OK`

```json
{
  "message": "Cluster renamed successfully",
  "clusterId": "uuid",
  "oldName": "my-cluster",
  "newName": "renamed-cluster"
}
```

## Error Responses

| Status | Description | Example |
|--------|-------------|---------|
| 400    | Missing project_id or invalid request | `{ "detail": "project_id required for non-api-key users" }` |
| 404    | Cluster not found | `{ "detail": "Cluster not found." }` |
| 409    | Duplicate cluster name | `{ "detail": "A cluster with this name already exists in the project." }` |
| 403    | Insufficient permissions | `{ "detail": "You do not have permission to rename this cluster." }` |
| 500    | Internal error | `{ "detail": "Failed to rename cluster: ..." }` |
