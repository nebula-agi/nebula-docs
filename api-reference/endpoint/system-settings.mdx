# System Settings

Get system settings and configuration information.

<RequestExample>
```bash
curl -X GET "https://api.r2r.ai/v3/system/settings" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

<ResponseExample>
```json
{
  "success": true,
  "data": {
    "app_name": "R2R Application",
    "version": "1.0.0",
    "environment": "production",
    "features": {
      "auth_enabled": true,
      "rate_limiting": true,
      "logging": true
    }
  }
}
```
</ResponseExample>

**Description:** Returns system settings and configuration information. Only accessible by superusers.

**Authentication:** Required (Superuser only)

**Response Fields:**
- `app_name` (string): Application name
- `version` (string): Application version
- `environment` (string): Environment name
- `features` (object): Feature flags and settings

## Error Codes

| Code | Description |
|------|-------------|
| `403` | Forbidden - Only superusers can access this endpoint |
| `401` | Unauthorized - Invalid or missing API key |
| `500` | Internal Server Error - System error |

## Rate Limiting

System settings: Rate limited

## SDK Examples

### Python

```python
from r2r import R2RClient

client = R2RClient(api_key="your-api-key")

# Get system settings
settings = client.system.settings()
print(f"App version: {settings['version']}")
```

### JavaScript

```javascript
const { r2rClient } = require("r2r-js");

const client = new r2rClient("your-api-key");

// Get system settings
const settings = await client.system.settings();
console.log(`App version: ${settings.version}`);
```
