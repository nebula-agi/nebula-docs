Get the status and basic info for a cluster. Requires authentication.

## Endpoint

`POST /clusters/status`

## Description
Returns the status and basic information for a cluster by name or ID in the authenticated user's project. Only the cluster owner or a project admin/collaborator can view the cluster.

## Request

**Headers:**
- `Authorization: Bearer <token>` or `Authorization: ApiKey <api_key>`

**Body:**
```json
{
  "cluster": "string" // Required. Cluster name or ID.
}
```

## Example cURL

```bash
curl -X POST https://api.nebulacloud.app/clusters/status \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "cluster": "my-cluster"
  }'
```

## Success Response

**Status:** `200 OK`

```json
{
  "id": "uuid",
  "name": "my-cluster",
  "status": "allocating",
  "description": "A test cluster",
  "projectId": "uuid"
}
```

## Error Responses

| Status | Description | Example |
|--------|-------------|---------|
| 400    | Missing project_id or invalid request | `{ "detail": "project_id required for non-api-key users" }` |
| 404    | Cluster not found | `{ "detail": "Cluster not found." }` |
| 403    | Insufficient permissions | `{ "detail": "You do not have permission to view this cluster." }` |
