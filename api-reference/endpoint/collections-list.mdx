# List Collections

Retrieve a list of collections accessible to the authenticated user.

<RequestExample>
```bash
curl -X GET "https://api.r2r.ai/v3/collections?offset=0&limit=10&owner_only=false" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

<ResponseExample>
```json
{
  "success": true,
  "data": {
    "results": [
      {
        "id": "123e4567-e89b-12d3-a456-426614174000",
        "name": "My Collection",
        "description": "A sample collection",
        "owner_id": "789a012b-c34d-5e6f-g789-012345678901",
        "created_at": "2024-01-15T10:30:00Z",
        "updated_at": "2024-01-15T10:30:00Z"
      }
    ],
    "total": 1,
    "offset": 0,
    "limit": 10
  }
}
```
</ResponseExample>

**Description:** Returns a paginated list of collections that the authenticated user has access to. Users can see collections they own and collections they are members of.

**Authentication:** Required

**Query Parameters:**
- `ids` (array of strings, optional): A list of collection IDs to retrieve. If not provided, all accessible collections will be returned.
- `offset` (integer, optional, default: 0): Number of objects to skip. Must be >= 0.
- `limit` (integer, optional, default: 100): Limit on the number of objects to return. Must be between 1 and 1000.
- `owner_only` (boolean, optional, default: false): If true, only returns collections owned by the user, not all accessible collections.

**Response Fields:**
- `results` (array): Array of collection objects
  - `id` (string): Unique identifier for the collection
  - `name` (string): Collection name
  - `description` (string): Collection description
  - `owner_id` (string): ID of the collection owner
  - `created_at` (string): ISO 8601 timestamp when the collection was created
  - `updated_at` (string): ISO 8601 timestamp when the collection was last updated
- `total` (integer): Total number of collections
- `offset` (integer): Current offset
- `limit` (integer): Current limit

## Error Codes

| Code | Description |
|------|-------------|
| `400` | Bad Request - Invalid query parameters |
| `401` | Unauthorized - Invalid or missing API key |
| `500` | Internal Server Error - System error |

## SDK Examples

### Python

```python
from r2r import R2RClient

client = R2RClient(api_key="your-api-key")

# List collections
result = client.collections.list(
    offset=0,
    limit=10,
    owner_only=False
)
print(f"Found {result['total']} collections")
```

### JavaScript

```javascript
const { r2rClient } = require("r2r-js");

const client = new r2rClient("your-api-key");

// List collections
const response = await client.collections.list({
    offset: 0,
    limit: 10,
    ownerOnly: false
});
console.log(`Found ${response.total} collections`);
```
