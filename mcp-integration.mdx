# MCP Integration
 
Connect Claude Desktop, Cursor, and VS Code–based clients to Nebula via MCP to store and search memories directly from the assistant.

<Info>Prefer <b>Node.js</b> for the simplest, fastest setup; use Python only if it fits an existing environment.</Info>

## Quick start
<Tabs>
  <Tab title="Claude Desktop">
  
  1) Run this one‑command installer:
  
  ```bash
  npx install-mcp "@nebula-ai/mcp-server --stdio" --client claude --env NEBULA_API_KEY=your_api_key
  ```
  
  2) Restart Claude Desktop and confirm Nebula tools appear.
  
  3) Config file paths if needed: macOS `~/Library/Application Support/Claude/claude_desktop_config.json`, Windows `%APPDATA%\\Claude\\claude_desktop_config.json`, Linux `~/.config/claude/claude_desktop_config.json`.
  
  </Tab>
  <Tab title="Cursor">
  
  1) One‑command install (project‑scoped or global):
  
  ```bash
  npx install-mcp "@nebula-ai/mcp-server --stdio" --client cursor --env NEBULA_API_KEY=your_api_key
  ```
  
  2) Config paths: Project `.cursor/mcp.json`, Global `~/.cursor/mcp.json`.
  
  3) Restart Cursor and check MCP tools in settings.
  
  </Tab>
  <Tab title="VS Code / Cline">
  
  1) One‑command install:
  
  ```bash
  npx install-mcp "@nebula-ai/mcp-server --stdio" --client cline --env NEBULA_API_KEY=your_api_key
  ```
  
  2) Cline config file: `~/.cline/mcp.json` (project overrides supported).
  
  3) VS Code supports direct MCP install and workspace `mcp.json` when preferred.
  
  </Tab>
</Tabs>

### Verify with Inspector
Use MCP Inspector to confirm auth, tools, and health before troubleshooting.

```bash
# Node server under Inspector
npx @modelcontextprotocol/inspector npx @nebula-ai/mcp-server --stdio --api-key your_api_key
```

If the browser UI loads and the tools are visible/invokable, the server is correctly configured.

## Environment
- NEBULA_API_KEY (required): API key for Nebula.
- NEBULA_API_URL (optional): Custom API endpoint, defaults to `https://api.nebulacloud.app`.

## Tools provided
- add_memory: Store docs or messages in a cluster.
- search_memories: Retrieve relevant items via natural language.
- health_check: Validate key and connectivity.

## Usage pattern
- Create or select a cluster, then use `add_memory` for saves and `search_memories` for retrieval during chats.

<Details>
  <Summary>Manual config (Node)</Summary>
  
  1) Optional global install: `npm install -g @nebula-ai/mcp-server`.
  
  2) Add to your client’s `mcpServers` JSON with stdio and env:
  
  ```json
  {
    "mcpServers": {
      "nebula": {
        "command": "npx",
        "args": ["-y", "@nebula-ai/mcp-server", "--stdio"],
        "env": {
          "NEBULA_API_KEY": "your_api_key_here",
          "NEBULA_API_URL": "https://api.nebulacloud.app"
        }
      }
    }
  }
  ```
  
</Details>

<Details>
  <Summary>Python option</Summary>
  
  For Python environments, install and run via `python -m`, keeping stdio transport in client config.
  
  ```bash
  pip install nebula-mcp
  npx install-mcp "python -m nebula_mcp" --client claude --env NEBULA_API_KEY=your_api_key
  ```
  
  Use the same `mcpServers` shape with `command: "python"`, `args: ["-m", "nebula_mcp"]`.
  
</Details>

<Details>
  <Summary>VS Code / Visual Studio one‑click</Summary>
  
  VS Code supports installing MCP servers directly from the web or the MCP Servers view; add an “Install in VS Code” badge to prefill the config.
  
  Visual Studio supports a deep‑link installer (`vsweb+mcp:/install?<encoded_json>`) for stdio/http servers; add an “Install in Visual Studio” badge.
  
</Details>
